## Cómo se generan/actualizan estos enlaces

Los links usan el servidor público de PlantUML (`plantuml.com/plantuml`), codificando el contenido
del `.puml` con `deflate` + un base64 propio de PlantUML. Si editas un `.puml`, regenera su link así
(no requiere instalar PlantUML, solo Python):

```python
import zlib
ALPHABET = "0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz-_"
def encode6bit(b): return ALPHABET[b & 0x3F]
def append3bytes(b1, b2, b3):
    c1, c2 = b1 >> 2, ((b1 & 0x3) << 4) | (b2 >> 4)
    c3, c4 = ((b2 & 0xF) << 2) | (b3 >> 6), b3 & 0x3F
    return "".join(encode6bit(c) for c in (c1, c2, c3, c4))
def encode(text: str) -> str:
    raw = text.encode("utf-8")
    co = zlib.compressobj(9, zlib.DEFLATED, -15)
    data = co.compress(raw) + co.flush()
    return "".join(append3bytes(data[i], data[i+1] if i+1<len(data) else 0, data[i+2] if i+2<len(data) else 0)
                   for i in range(0, len(data), 3))

print("https://www.plantuml.com/plantuml/svg/" + encode(open("mi-diagrama.puml", encoding="utf-8").read()))
```

Alternativas para verlos/editarlos sin depender de un link externo:
- **VS Code**: extensión "PlantUML" (jebbs) — abre el `.puml` y `Alt+D` previsualiza en el editor.
- **IntelliJ / otros IDE JetBrains**: plugin "PlantUML integration".
- **CLI local**: `plantuml *.puml` (requiere Java + `plantuml.jar`, genera `.png`/`.svg` locales).
- **Editor online**: pega el contenido del `.puml` en https://www.plantuml.com/plantuml/uml/ .

## Contenido de cada diagrama

- **`00-contexto.puml`** — quién usa el sistema (jugadores, anfitriones) y con qué sistemas externos

## Notas de fidelidad

- Los diagramas reflejan el código tal como está en `main` al momento de generarlos. Si el código
  cambia (nueva clase, nuevo campo relevante), el `.puml` correspondiente debe actualizarse a mano.
- El diagrama de despliegue documenta explícitamente que la VM corre `docker-compose.backend.yml`
  (sin frontend ni stack de observabilidad) — **no** `docker-compose.yml` completo — para no dar una
  imagen falsa de lo que hay corriendo hoy en Azure.
- `voice-signaling-service` y `api-gateway` tienen diagramas de clases más simples a propósito: el
  primero no tiene capas domain/app (ADR-010, es un relay sin modelo de dominio) y el segundo es
  configuración declarativa (rutas en YAML) más 2-3 clases de código.
